使用 hexo 搭建博客

发表于 2019-04-16 更新于 2020-03-17 Disqus：

1.基本步骤

可以参考 https://hexo.io/docs/index.html 上的官方文档来搭建环境。这里总结一下主要步骤，首先是安装 nodejs 和 npm 包管理器，然后通过包管理器安装 hexo。

sudo apt install nodejs         # 安装 nodejs
sudo apt install npm            # 安装 npm 包管理器，可省略
sudo npm install -g hexo-cli    # 全局安装 Hexo

Node.js 是一个基于 Chrome JavaScript V8 引擎的 JavaScript 语言的运行时库。我们使用的网页浏览器通常会内置 JavaScript 的解释器，所以网页中的 JavaScript 代码可以被浏览器解释和执行。但是如果需要脱离浏览器环境运行，比如需要在本地开发 JavaScript 应用的话，就可以安装和使用 Node.js 这个运行时库，它能够代替浏览器提供程序的运行环境。hexo 是基于 Node.js 的包，所以需要先安装 Node.js。

npm 是一个 Node.js 的包管理器，可以让 JavaScript 的开发者方便的共享和重用代码，有点类似于 ubuntu 的 apt 工具。npm 是随着 Node.js 一起发布的，所以一般来说，安装了 Node.js 也就安装了 npm（不过我们上面还是单独执行了 apt install npm）。

用户可以通过 npm install 来安装包，安装分为全局安装和本地安装，区别是是否带有参数 -g。本地安装会把包安装在 ./node_modules 目录，而全局安装则会安装在 /usr/local 或者 node 的安装目录中。上面的代码通过 npm install -g hexo-cli 安装了 hexo 博客系统这个包。

对于一个新的目录，可以进行如下初始化。hexo init 会创建 hexo 工程的一些必要文件（比如后面 npm install 所需要的 package.json），而 npm install 会根据生成的目录中的 package.json 文件来安装各种依赖包。

$ hexo init <folder>   # 在目录中创建 hexo 环境
$ cd <folder>
$ npm install          # 根据 package.json 在目录中建立和填充 node_modules 目录，是 hexo 需要使用的各个支持包

初始化完成之后的目录结构如下。source 目录用来存放 markdown 写的原始文章，之后通过 hexo 命令把他们转化成网页。themes 用来存放 hexo 主题，下载的其他人共享的主题目录就是放在这个目录中。_config.yml 配置文件可以对站点进行配置，比如选择使用 theme 目录中的哪个主题。

node_modules\
public\         # 这个目录是 hexo g 生成的网站所在的目录，初始的时候这个目录不存在
scaffolds\
source\         # .md 文章存放的目录
themes\         # 主题模板存放目录
    theme1\
    theme2\
_config.yml
package.json

安装完了之后可以安装新的主题模板（这部分不详细说，请参考官网的步骤来选择和安装），以及修改 _config.yml 文件，完成本地适配工作。至于写文章，主要步骤如下

  • 使用 hexo new “文章名字” 来新建新的文档，也可以在 sources/_posts/ 目录下直接新建 .md 文章
  • 使用 hexo g 读取 source 目录下的用户文章，生成 public 目录的内容，这个也是网页最终呈现的内容
  • 使用 hexo s 可以在本地预览站点，站点的根目录也就是 public 目录。浏览器会读取 public 目录中的内容并显示

2.github pages

上一节说到了如何在本地环境下创建 hexo 环境、利用 hexo 写文章、生成网站并且进行本地预览。不过我们希望把个人网站放到公网上给别人访问，github pages 可以帮助我们达到这个目的。github pages 是 github 公司提供的免费的静态网页托管服务，它对应于用户的一个特殊的仓库，这个仓库的名字必须是 username.github.io，其中 username 是用户的 github 用户名。

hexo 能够生成适用于 github pages 的静态网站，我们只需要把 hexo 生成的 public 目录中的内容上传到仓库的 master 分支就可以了。注意不要将网站放到其它分支，因为 github pages 只会呈现 master 分支的内容。

3.环境迁移

为了方便写作环境的迁移，比如换了一台电脑写作，最好也把 hexo 的开发环境也保存在 github 中。具体做法是把 public 目录放在 master 分支中，而 hexo 开发目录放在分支中，比如 develop 分支中。

develop 分支中的目录结构如下，分支的内容就是前面章节生成的完整的 hexo 开发目录。需要注意的是，最好不要把 node_modules 目录上传到 git 仓库中，因为这个是和系统相关的，如果换了一个环境还是使用原来的 node_modules，很可能碰到一些奇怪的问题。

scaffolds\
source\     # source 目录中存放了 .md 后缀的文章
theme\
\_config.yml
package.json
package-lock.json

master 分支的目录结构如下，其实就是通过 hexo s 命令生成的 public 目录中的内容

about\
archives\
css\
images\  
index.html  
js\
...

通常我选择 clone 仓库到两个不同的目录，分别切换为 master 分支和 develop 分支。在 develop 分支进行文章的撰写并且生成 public 目录，然后通过目录比较工具（比如 beyond compare）或者直接拷贝 public 目录中的内容到 master 分支，替换掉目录中原来的内容，再上传 master 分支就行了。如下图所示，master 分支是 develop 分支 public 中的内容。

+--------------------------+        +-------------------------+
|username.github.io develop|        |username.github.io master|
|                          |        |                         |
|node_modules\             |        |                         |
|source\                   |        |                         |
|public\ -------------------->copy----> content of public dir |
|theme\                    |        |                         |
|...                       |        |                         |
+--------------------------+        +-------------------------+

如果需要在一台新的电脑上进行写作，可以按照以下步骤进行，首先就是同样需要在新的电脑上安装 hexo 的支持环境，这个是必不可少的

sudo apt install nodejs         # 安装 nodejs
sudo apt install npm            # 安装 npm 包管理器
sudo npm install -g hexo-cli    # 全局安装 Hexo

然后可以把 github 上面的代码 clone 下来，并且切换到 develop 分支，安装本地系统需要的 node_modules，然后就可以进行写作了。

git clone username.github.io  # clone 用户自己的 github pages 仓库，注意把 username 替换成自己的仓库名
git checkout develop          # 切换到 develop 分支
npm install                   # 安装系统需要的 node_modules

4.图片显示

图片显示是 hexo 比较麻烦的地方。一种方法是使用外部链接，比如把图片放在支持图片外链云盘中，文章中使用云盘中的图片链接。另外一种办法是将图片也保存在 hexo 站点中，hexo 提供了 post_asset_folder 选项，这个选项可以在创建文章的时候创建一个和文章名字同名的文件夹，图片可以放在这个目录中，只需要修改 _config.yml 文件，设置 post_asset_folder: true 就可以了。

post_asset_folder: true    # 在 _config.yml 目录中

在这之后，如果需要在文章中引用图片，可以使用下面的语法

{% asset_img "image" "image desc" %}           // 显示图片
{% asset_img "image" xxx xxx "image desc" %}   // 显示图片，设置图片的宽高参数，可以只设置宽，那么高是自动计算的

5.语法高亮

hexo 使用的高亮插件不支持对代码中的特殊语句进行高亮，比如如下代码

code  // 注释1
code  // 注释2

想对其中的 注释1 单独高亮是不支持的。我的想法是在 markdown 文章中直接插入 html 代码块，在特定的需要高亮的地方有插入 span 标签，然后在 css 中针对 sapn 设置格式。一段插入的 html 代码如下所示，可以设置 pre.themepre 和 span.themespan 的 css 属性，让 html 段呈现比较好的效果。

<pre class="themepre">
code  <span class="themespan">// 注释1</span>
code  // 注释2
</pre>

采用上面直接插入 html 块的方式显示代码，会发现一些代码出现乱码或者显示错误，究其原因就是 hexo-renderer-marked 包在处理 markdown 页面的时候，不会对 html 标签中的任何内容做处理，而由浏览器自行解析，比如下面的一段代码

&para_start_addr_1;
<code></code>

浏览器解析的结果如下，一些 html 的标记被浏览器“正确的”解析了（其中的 &para_start_addr_1，虽然不符合 &para; 但是浏览器还是变成了特殊符号）。

¶_start_addr_1;
//此处为空行，因为浏览器解析 <code> 为标签

hexo 处理 markdown 文章的时候是直接调用的 node_modules/hexo-renderer-marked 这个模块进行处理的，而其中的 renderer.js 又是调用的 node_modules/marked 进行的解析。对 node_modules/hexo-renderer-marked/lib/renderer.js 中的部分代码做修改，添加对于 html block 段的特殊处理，只保留用于高亮的特殊 html 标记，其他的 html 标记全部做转换。这段修改是根据 html 段以 <pre class="themepre"> 开头来做匹配的，如果仍然想 html 块中的内容被按照 html 来解析，只需要让 html 段不符合匹配格式即可，比如修改 html 段为以 <pre class="themepre" class="undo"> 开头。

这段代码只是做了简单处理，所以还是会有一些情况，比如代码中有 </span> 会显示不了，不过关系不是太大。

// node_modules/hexo-renderer-marked/lib/renderer.js
Renderer.prototype.html = function(text) {
  var result;
  // 这里以 <pre class="themepre"> 开头作为匹配条件
  // 如果不想使用这段配置的函数，也就是仍然想其中的内容被作为 html 解析，可以简单的对 html 标签做修改，比如改为 <pre class="themepre" class="undo">
  if (/^<pre class="themepre">/.test(text)) {
    // 首先替换掉 &, <, > 等特殊符号
    text = text.replace(/&/g, "&amp;").replace(/</g, '&lt;').replace(/>/g, '&gt;').replace(/"/g, '&quot;').replace(/'/g, '&#39;');

    // 对于两个高亮的 html 语句，再替换回来
    text = text.replace(/&lt;pre class=&quot;themepre&quot;&gt;/g, "<pre class=\"themepre\">");
    text = text.replace(/&lt;\/pre&gt;/g, "</pre>");

    text = text.replace(/&lt;span class=&quot;themespan&quot;&gt;/g, "<span class=\"themespan\">");
    text = text.replace(/&lt;\/span&gt;/g, "</span>");

    result = text;
  } else {
    result = text;
  }

  return result;
};
